# entities

Business knowledge, NOT work. Customers, competitors, prospects, content calendar, positioning
notes — markdown, one file per entity or per set. Beads track *work*; this tracks *things the
work is about*. Never force these into the issue tracker.

Examples: `competitors.md`, `icp-interviews.md`, `content-calendar.md`, `prospects.md`.

A `domain:marketing`/`domain:bizdev` bead's `## Context` can point here (e.g. "draft outreach
to the 5 prospects in `.product/entities/prospects.md`").
