# decisions

ADR-style notes. One file per decision, referenced from a bead's `## Context` (e.g.
`.product/decisions/pricing.md#tiers`). Keep them short: the decision, the why, the
alternatives rejected. Not a changelog — only decisions a future shaper or worker needs.

`YYYY-MM-DD-<slug>.md`:

```md
# <decision>
Date: YYYY-MM-DD
Status: accepted | superseded by <file>

## Decision
<what we decided>

## Why
<the reasoning; the constraint that forced it>

## Rejected
<alternatives and why not>
```
